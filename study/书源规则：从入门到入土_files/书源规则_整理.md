## 1、语法说明

### JSoup 之 Default

```
@为分隔符,用来分隔获取规则
·每段规则可分为3段
·第一段是类型,如class,id,tag,text,children等
  - children获取所有子标签,不需要第二段和第三段
  - text可以根据文本内容获取
·第二段是名称,text.第二段为文本内容的一部分
·第三段是位置,class,tag,id等会获取到多个,所以要加位置
·如不加位置会获取所有
·位置正着数从0开始,0是第一个,如为负数则是取倒着数的值,-1为倒数第一个,-2为倒数第二个
·!是排除,有些位置不符合需要排除用!,后面的序号用:隔开0是第1个,负数为倒数序号,-1最后一个,-2倒数第2个,依次
·获取列表的最前面加上负号-可以使列表倒置
·@的最后一段为获取内容,如text,textNodes,ownText,href,src,html,all等
·如需要正则替换在最后加上 ##正则表达式##替换内容，替换内容为空时，第二个##可以省略
```

**示例：**

```text
class.odd.0@tag.a.0@text||tag.dd.0@tag.h1@text##全文阅读
class.odd.0@tag.a.0@text&&tag.dd.0@tag.h1@text##全文阅读
```

**数组写法：**

```text
格式：[index,index,...]或[!index,index,...]
区间格式：[start:end]或[start:end:step]
特殊用法：tag.div[-1:0] 可让列表反向
```

**等价写法：**

```text
head@.1@text == head@[1]@text == head@children[1]@text
```

**参考文档：**
- [JSoup Selector](https://jsoup.org/apidocs/org/jsoup/select/Selector.html)

### JSoup 之 CSS

- 必须以 `@css:` 开头
- 语法参考：[CSS Selector 教程](https://blog.csdn.net/hou_angela/article/details/80519718)

```text
获取内容可用：text, textNodes, ownText, html, all, href, src 等
```

### JSONPath

- 最好以 `@json:` 或 `$.` 开头
- 标准规范：[goessner JSONPath](https://goessner.net/articles/JsonPath/)
- 实现库：[json-path/JsonPath](https://github.com/json-path/JsonPath)
- 在线测试：[Jayway JsonPath Evaluator](http://jsonpath.herokuapp.com/)

---

### XPath

- 必须以 `@XPath:` 或 `//` 开头
- 标准规范：[W3C XPATH 1.0](https://www.w3.org/TR/1999/REC-xpath-19991116/)
- 实现库：[zhegexiaohuozi/JsoupXpath](https://github.com/zhegexiaohuozi/JsoupXpath)

---

### JavaScript

- 可以在 `<js></js>`、`@js:` 中使用，结果存在 `result` 中
- `@js:` 只能放在其他规则的最后使用
- `<js></js>` 可以在任意位置使用，还能作为其他规则的分隔符

```text
例：tag.li<js></js>//a
```

---

### 正则之 AllInOne

- 只能在搜索列表、发现列表、详情页预加载和目录列表中使用
- 必须以 `:` 开头

---

### 正则之 OnlyOne

```text
形式：##正则表达式##替换内容###
```

- 只能获取第一个匹配到的结果并进行替换

---

### 正则之净化

```text
形式：##正则表达式##替换内容
```

- 只能跟在其他规则后面，独立使用相当于 `all##正则表达式##替换内容`
- 该规则为循环匹配替换

---

### 连接符号

| 符号 | 行为 |
|------|------|
| `&&` | 合并所有取到的值 |
| `\|\|` | 以第一个取到值的为准 |
| `%%` | 依次取数，如三个列表，先取列表1的第一个，再取列表2的第一个... |

## 2、Legado的特殊规则

### URL 必知必会

#### 请求头

**一般形式：**

```json
{
    "User-Agent": "Mozilla/5.0...",
    "Accept-Language": "zh-CN,zh;q=0.9"
}
```

**复杂情况可使用 JS：**

```javascript
(() => {
    var ua = "Mozilla/5.0...";
    var headers = {"User-Agent": ua};
    return JSON.stringify(headers);
})()
```

#### 获取登录后的 Cookie

```javascript
java.getCookie("http://baidu.com", null)  // => userid=1234;pwd=adbcd
java.getCookie("http://baidu.com", "userid")  // => 1234
```

#### 代理设置

```json
// socks5代理
{"proxy": "socks5://127.0.0.1:1080"}

// http代理
{"proxy": "http://127.0.0.1:1080"}

// 带用户名密码
{"proxy": "socks5://127.0.0.1:1080@用户名@密码"}
```

#### GET 请求

**一般形式：**

```text
<https://www.baidu.com,{
    "charset": "gbk",
    "headers": {"User-Agent": "..."},
    "webView": true
}>
```

**复杂情况使用 JS：**

```javascript
var option = {
    "charset": "gbk",
    "headers": headers,
    "webView": true
};
"https://www.baidu.com," + JSON.stringify(option)
```

#### POST 请求

**一般形式：**

```text
<https://www.baidu.com,{
    "charset": "gbk",
    "method": "POST",
    "body": "bid=10086",
    "headers": {"User-Agent": "..."},
    "webView": true
}>
```

### @put 与 @get

只能用于 JS 以外的规则中

```text
@put:{bid:"//*[@bid-data]/@bid-data"}
```

---

### {{}} 与 {} 规则

**在搜索URL与发现URL中的 `{{}}`：**
- 在 `{{}}` 里只能使用 JS

**在搜索URL与发现URL以外的 `{{}}`：**
- 可在 `{{}}` 中使用任意规则（正则除外）
- 默认为 JS，使用其他规则需要有明显的标志头：
  - Default 规则：`@@` 开头
  - XPath：`@xpath:` 或 `//` 开头
  - JSONPath：`@json:` 或 `$.` 开头
  - CSS：`@css:` 开头

**`{}` 规则：**
- 留用了阅读2.0的规则，只能使用 JSONPath

## 4、书源之「搜索」

### searchUrl

```text
{{key}} = 搜索关键字
{{page}} = 页码（初值为1）

// 示例
/search?key={{key}}&page={{page}}

// key加密
{{java.base64Encode(key)}}

// page计算
{{(page-1)*20}}

// 第一页无页数的情况
① {{page - 1 == 0 ? "" : page}}
② <,{{page}}>
```

## 6、书源之「详情页」

### ruleBookInfo

| 字段 | 说明 |
|------|------|
| `bookInfoInit` | 预处理规则（只能用正则AllInOne或JS） |
| `name` | 书名规则 |
| `author` | 作者规则 |
| `kind` | 分类规则 |
| `wordCount` | 字数规则 |
| `lastChapter` | 最新章节规则 |
| `intro` | 简介规则 |
| `coverUrl` | 封面规则 |
| `tocUrl` | 目录URL规则（只支持单个url） |
| `canReName` | 允许修改书名作者 |

**bookInfoInit 示例（JS）：**

```javascript
(function(){
    return {
        a: "圣墟",           // 书名规则填 a
        b: "辰东",           // 作者规则填 b
        c: "玄幻",           // 分类规则填 c
        d: "200万字",        // 字数规则填 d
        e: "第两千章...",     // 最新章节填 e
        f: "简介内容...",     // 简介填 f
        g: "https://...",    // 封面填 g
        h: "https://..."     // 目录URL填 h
    };
})()
```

## 9、补充说明

### 显示 JS 报错信息

```javascript
(function(result){
    try{
        // 处理result
        return result;  // 字符串返回
        return list;   // 列表返回
    }
    catch(e){
        return ""+e;       // 字符串错误
        return [""+e];     // 列表错误（对应名称处填 <js>""+result</js>）
    }
})(result);
```

| 调试项 | 输入示例 |
|--------|----------|

## 书源示例

### 书源一（CSS + 正则）

```json
{
    "bookSourceName": "小说2016",
    "bookSourceUrl": "https://www.xiaoshuo2016.com",
    "ruleSearch": {
        "author": "@css:p:eq(2)>a@text",
        "bookList": "@css:li.clearfix",
        "name": "@css:.name@text"
    },
    "ruleToc": {
        "chapterList": "-:<li><a[^\"]+\"([^\"]_)\">([^<]_)",
        "chapterName": "$2",
        "chapterUrl": "$1"
    },
    "searchUrl": "/modules/article/search.php?searchkey={{key}}&submit=&page={{page}},{\n  \"charset\": \"gbk\"\n}"
}
```

### 书源二（XPath + 正则）

```json
{
    "bookSourceName": "采墨阁手机版",
    "bookSourceUrl": "https://m.caimoge.com",
    "ruleBookInfo": {
        "author": "//*[@property=\"og:novel:author\"]/@content",
        "name": "//*[@property=\"og:novel:book_name\"]/@content"
    },
    "ruleSearch": {
        "author": "//dd[2]/text()",
        "bookList": "//*[@id=\"sitebox\"]/dl",
        "name": "//h3/a/text()"
    },
    "searchUrl": "/search.html,{\n  \"method\": \"POST\",\n  \"body\": \"searchkey={{key}}\"\n}"
}
```
