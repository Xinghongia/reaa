# 第三章：业务逻辑层 - 学习笔记与问答

## 本章学习内容

### 3.1 为什么需要业务逻辑层？

```
┌─────────────────────────────────────────────────────────────┐
│                   没有业务逻辑层的问题                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  UI 层直接调用数据层                                         │
│       ↓                                                     │
│  屏幕旋转 → 重新请求网络 → 慢！卡！                         │
│       ↓                                                     │
│  多个 UI 使用同一份数据 → 数据不一致！                       │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                   有业务逻辑层的好处                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  UI 层 → ViewModel（业务逻辑）→ Repository → DAO          │
│                ↓                                           │
│           状态管理、缓存、数据转换                           │
│                ↓                                           │
│  屏幕旋转 → ViewModel 保留 → UI 立即恢复                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 规则解析 (analyzeRule)

#### 为什么需要多种解析器？

```
┌─────────────────────────────────────────────────────────────┐
│                   多种解析器的原因                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  不同网站的 HTML 结构不同                                   │
│       ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 网站A：<div class="book"><a href="...">书名</a>    │   │
│  │ 网站B：<ul id="list"><li><a href="...">书名</a>    │   │
│  │ 网站C：<table><tr><td>1</td><td>书名</td></tr>     │   │
│  └─────────────────────────────────────────────────────┘   │
│       ↓                                                     │
│  同一个解析器无法处理所有网站                               │
│                                                             │
│  ─────────────────────────────────────────────────────    │
│                                                             │
│  解决方案：策略模式                                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ RuleAnalyzer (统一入口)                              │   │
│  │      ↓                                              │   │
│  │   ├── AnalyzeByXPath   (XML/HTML 结构化)           │   │
│  │   ├── AnalyzeByJSoup   (CSS Selector)              │   │
│  │   ├── AnalyzeByRegex   (文本匹配)                  │   │
│  │   └── AnalyzeByJSonPath (JSON API)                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 解析器对比

```
┌─────────────────────────────────────────────────────────────┐
│                   解析器对比                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┬──────────────┬─────────────────────────┐  │
│  │   解析器      │   使用场景    │        示例             │  │
│  ├──────────────┼──────────────┼─────────────────────────┤  │
│  │ XPath        │ HTML结构清晰  │ //div[@class="book"]    │  │
│  │ JSoup        │ 需要JS执行    │ .book-list a           │  │
│  │ Regex        │ 文本不规则    │ 书名：([^，]+)          │  │
│  │ JSONPath     │ API返回JSON  │ $.data.books[*].title   │  │
│  └──────────────┴──────────────┴─────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 阅读器核心 (ReadBook)

#### ReadBook 核心流程

```
┌─────────────────────────────────────────────────────────────┐
│                   ReadBook 核心流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  打开书籍                                                   │
│      ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ LoadContent (加载章节内容)                          │   │
│  │ 1. 检查本地缓存                                    │   │
│  │ 2. 网络获取                                        │   │
│  │ 3. 内容净化 (ReplaceRule)                         │   │
│  │ 4. 缓存持久化                                      │   │
│  └─────────────────────────────────────────────────────┘   │
│      ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Paginate (分页)                                    │   │
│  │ 根据屏幕大小、字体、行间距计算页数                 │   │
│  └─────────────────────────────────────────────────────┘   │
│      ↓                                                     │
│  RenderPage (渲染页面)                                      │
│      ↓                                                     │
│  用户翻页 → SaveProgress (保存进度)                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 分页算法原理

```
┌─────────────────────────────────────────────────────────────┐
│                     分页算法原理                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  每页可显示字数 = 屏幕宽度 ÷ 字符宽度 × 屏幕高度 ÷ 行高     │
│                                                             │
│  例：屏幕 1080×1920，字体 16sp，行高 1.5倍                   │
│  → 每行约 67 字 (1080÷16)                                   │
│  → 每屏约 120 行 (1920÷16÷1.5)                              │
│  → 每页约 8000 字                                          │
│                                                             │
│  ─────────────────────────────────────────────────────    │
│                                                             │
│  内容 "第一章XXXXXXXX..." (50000字)                          │
│       ↓                                                     │
│  50000 ÷ 8000 = 6.25 → 7 页                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────┐
```

### 3.8 设计模式总结

```
┌─────────────────────────────────────────────────────────────┐
│                   业务逻辑层设计模式                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 策略模式 - 规则解析器                                    │
│     RuleAnalyzer 根据网站类型选择 XPath/JSoup/Regex/JSON   │
│                                                             │
│  2. 工厂模式 - 本地书籍解析                                  │
│     LocalBook 根据扩展名创建 TextFile/EpubFile/PdfFile     │
│                                                             │
│  3. 模板方法 - 解析基类                                     │
│     BaseLocalBookParse 定义统一接口，子类实现具体解析        │
│                                                             │
│  4. 责任链模式 - 内容净化                                   │
│     ReplaceRule 链式处理，多条规则依次应用                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
