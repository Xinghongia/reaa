# 第一章：项目概览 - 学习笔记与问答

## 本章学习内容

### 1.1 项目定位

**Legado with MD3** 是 Android 阅读器应用，是 [Legado](https://github.com/gedoor/legado) 的 Material Design 3 重写版本。

#### 核心能力

| 功能 | 说明 |
|------|------|
| 本地阅读 | 支持 TXT、EPUB、PDF、UMD 格式 |
| 在线书源 | 通过规则解析获取网络小说 |
| RSS 订阅 | 订阅 RSS/Atom 源获取更新 |
| 书籍缓存 | 离线缓存章节内容 |

### 1.2 技术栈

```
┌─────────────────────────────────────────────────────────────┐
│                        技术栈总览                            │
├─────────────────────────────────────────────────────────────┤
│  UI层     │ Jetpack Compose + Material 3                    │
│  DI       │ Koin（轻量级依赖注入）                          │
│  数据库    │ Room + KSP（编译时生成）                        │
│  网络     │ OkHttp + Cronet（Chromium 网络栈）               │
│  JS执行   │ Rhino（纯Java实现的JS引擎）                      │
│  Web服务器 │ NanoHTTPD（嵌入式HTTP服务器）                   │
│  图片加载  │ Coil + Glide                                   │
└─────────────────────────────────────────────────────────────┘
```

## 问答记录

## Q1: Koin 和 Hilt 是什么？

**Q**: 之前提到项目用 Koin 做依赖注入，Koin 和 Hilt 是什么？有什么区别？

**A**:

### 定义

| 框架 | 类型 | 维护方 |
|------|------|--------|
| **Hilt** | 依赖注入（DI）框架 | Google 官方 |
| **Koin** | 依赖注入框架 | 社区 |

### 核心区别

| 维度 | Hilt | Koin |
|------|------|------|
| **原理** | 编译时生成代码（注解处理器） | 运行时反射 |
| **性能** | 高 | 略低（但影响小） |
| **学习成本** | 高（注解多） | 低（DSL 风格） |
| **配置复杂度** | 复杂 | 简单 |

### 代码对比

**Hilt（Google 官方）**：
```kotlin
@HiltViewModel
class BookViewModel @Inject constructor(
    private val repository: BookRepository
) : ViewModel()
```

**Koin（社区主导）**：
```kotlin
val viewModelModule = module {
    viewModel { BookViewModel(get()) }
}
// 使用
val viewModel: BookViewModel by viewModel()
```

### 为什么 Legado 选择 Koin？

```
┌─────────────────────────────────────────────────────────────┐
│                  为什么选择 Koin？                          │
├─────────────────────────────────────────────────────────────┤
│  1. 项目规模适中，不需要 Hilt 的企业级特性                   │
│  2. Koin 配置简单，代码量少                                  │
│  3. 更符合 Kotlin 惯用风格                                  │
│  4. 从原版迁移成本低                                        │
└─────────────────────────────────────────────────────────────┘
```

### 适合场景

| 框架 | 适合项目 |
|------|----------|
| **Hilt** | 大型企业应用（Gmail、YouTube） |
| **Koin** | 中小型应用、快速开发、Kotlin First |

## Q2: Legado with MD3 相比原版 Legado 有什么新功能？

**Q**: 这个项目在原版 Legado 的基础上添加了什么功能？就只是重构了 UI？

**A**:

**核心变化：UI 层的现代化重构**，功能层面基本保持不变。

### 对比表

| 方面 | 原版 Legado | Legado with MD3 |
|------|-------------|------------------|
| **UI 框架** | 传统 Views + XML | Jetpack Compose |
| **设计语言** | Material 2 | Material Design 3 |
| **主题系统** | 静态主题 | Monet 动态颜色 |
| **状态管理** | LiveData | StateFlow + Compose |
| **导航** | FragmentManager | Navigation Compose |

### UI 迁移对比图

```
原版 Legado（Views）：
┌─────────────────────────────────────────┐
│  Activity / Fragment                    │
│  ├── XML Layout                         │
│  │   ├── TextView                       │
│  │   ├── RecyclerView                   │
│  │   └── Button                         │
│  └── ViewModel (LiveData)               │
└─────────────────────────────────────────┘
              ↓ 迁移
Legado MD3（Compose）：
┌─────────────────────────────────────────┐
│  Compose Activity                       │
│  ├── Compose UI                         │
│  │   ├── Text()                         │
│  │   ├── LazyColumn()                   │
│  │   └── Button()                       │
│  └── ViewModel (StateFlow)             │
└─────────────────────────────────────────┘
```

### 功能保持不变

```
┌─────────────────────────────────────────┐
│              核心功能                    │
├─────────────────────────────────────────┤
│  ✅ 本地书籍阅读（TXT/EPUB/PDF/UMD）     │
│  ✅ 在线书源解析                         │
│  ✅ RSS 订阅                            │
│  ✅ 书籍缓存下载                         │
│  ✅ 朗读功能                            │
│  ✅ Web 管理界面                        │
└─────────────────────────────────────────┘
```

### 总结

```
┌─────────────────────────────────────────────────────────────┐
│                        总结                                  │
├─────────────────────────────────────────────────────────────┤
│  Legado with MD3 = 原版 Legado 的 UI 现代化升级              │
│                                                             │
│  • 核心业务逻辑不变                                         │
│  • 界面层使用 Compose 重写                                  │
```
