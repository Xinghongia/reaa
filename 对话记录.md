## 一、项目构建问题解决

### 问题描述

执行 `./gradlew assembleDebug` 时构建失败，错误信息：

```
e: [ksp] java.lang.IllegalArgumentException:
this and base files have different roots:
C:\Users\33762\.gradle\caches\9.3.1\transforms\...\glide-svg-4.0.2-api.jar
and F:\2025python\legado-with-MD3\app.
```

### 问题原因

KSP 2.3.6 在 Windows 上处理 JAR 内部路径时存在 bug：
- Gradle 缓存路径使用正斜杠 `C:\...`
- 项目路径使用反斜杠 `F:\...`
- 路径比较失败

### 解决过程

1. **第一次尝试**：升级 KSP 版本
   - 将 `libs.versions.toml` 中 `ksp = "2.3.6"` 改为 `"2.3.10"`
   - 结果：版本不存在，失败

2. **第二次尝试**：移除 glide-svg
   - 从 `app/build.gradle.kts` 中删除 `implementation(libs.glide.svg)`
   - 结果：问题依旧，glide-okhttp 也触发了同样的 bug

3. **第三次尝试**：移除 glide-ksp
   - 从 `app/build.gradle.kts` 中删除 `ksp(libs.glide.ksp)`
   - 结果：**构建成功**

### 最终解决方案

移除两个无用依赖：
```kotlin
// app/build.gradle.kts 中移除
implementation(libs.glide.svg)  // 已移除
ksp(libs.glide.ksp)             // 已移除
```

### 依赖分析结论

- **glide-svg**：`SvgUtils.kt` 使用的是 `androidsvg` 库，不是 glide-svg
- **glide-ksp**：代码注释明确说使用原生 `com.bumptech.glide` API，不使用 `GlideApp`

   - 后台线程执行查询

   - 主键索引 → O(1) 查询
   - 复合索引 → 组合查询加速

#### 02_data_layer/03_repository/README.md - Repository

**源码位置**：`app/src/main/java/io/legado/app/data/repository/`

**核心内容**：

1. **为什么需要 Repository**
   - ViewModel 不直接依赖 DAO
   - 业务逻辑封装
   - 便于测试和替换实现

2. **Repository vs DAO 对比**

| 维度 | DAO | Repository |
|------|-----|-----------|
| 职责 | 数据存取 | 业务逻辑 + 数据存取 |
| 接口 | 数据库操作 | 领域操作 |
| 抽象 | 底层实现 | 用例驱动 |

3. **Koin 依赖注入配置**
   - `singleOf(::BookRepository)` 单例创建
   - 构造器注入 DAO

4. **完整数据流链路**
   ```
   Compose UI → ViewModel → Repository → DAO → Room Database → SQLite
   ```

5. **Flow → StateFlow 转换**
   - `stateIn(viewModelScope, ...)` 转换
   - `SharingStarted.WhileSubscribed(5000)` 保留旧数据5秒

#### 03_model_layer/01_analyze_rule/README.md - 规则解析

**源码位置**：`app/src/main/java/io/legado/app/model/analyzeRule/`

**核心内容**：

1. **RuleAnalyzer 入口设计**
   - Facade 模式统一入口
   - 策略模式选择解析器
   - 适配器模式统一返回格式

2. **四种解析器**

| 解析器 | 适用场景 | 示例 |
|--------|---------|------|
| XPath | HTML 结构清晰 | `//div[@class="title"]` |
| JSoup | 需要执行 JS | CSS Selector |
| Regex | 结构不规则文本 | `书名：([^"]+)` |
| JSONPath | API 返回 JSON | `$.data.books[*]` |

3. **JS 增强解析原理**
   - Rhino 引擎执行 JS
   - 处理懒加载、加密参数、登录态

4. **书源规则流程**
   ```
   searchUrl → 搜索页 → 书籍列表
   catalogUrl → 目录页 → 章节列表
   contentPattern → 正文页 → 章节内容
   ```

#### 03_model_layer/02_read_book/README.md - 阅读器核心

**源码位置**：`app/src/main/java/io/legado/app/model/ReadBook.kt`

**核心内容**：

1. **内容加载流程**
   ```
   检查本地缓存 → 网络获取 → 内容净化 → 缓存
   ```

2. **分页算法原理**
   - 根据屏幕大小和字体计算
   - 考虑字符宽度（中英文不同）
   - 按段落分割

3. **进度保存机制**
   - 保存时机：翻页、退出、切换章节
   - 保存内容：`chapterIndex`、`charIndex`、`scrollOffset`
   - 精确到字符位置

4. **净化规则应用**
   - 按优先级排序
   - 支持普通替换、正则替换、首匹配替换

5. **预加载 + LRU 缓存**
   - 当前章节 ±2 预加载
   - LRU 淘汰策略

#### 03_model_layer/03_local_book/README.md - 本地书籍解析

**源码位置**：`app/src/main/java/io/legado/app/model/localBook/`

**核心内容**：

1. **LocalBook 工厂模式**
   - 根据扩展名选择解析器
   - 统一入口 `LocalBook.parse(file)`

2. **TXT 解析**
   ```
   文件流 → 编码检测 → 文本读取 → 正则目录识别 → 分章
   ```

3. **TxtTocRule 目录规则**
   ```kotlin
   // 示例规则
   pattern = "第[0-9]+章.*"     // 匹配 "第一章"
   pattern = "\\[第[0-9]+章.*"  // 匹配 "[第123章"
   ```

4. **EPUB 解析**
   ```
   EPUB文件(ZIP)
   → 解压 → container.xml → content.opf
   → 解析manifest → 构建目录 → 按spine读取章节
   ```

5. **PDF 解析限制**
   - PdfRenderer 不支持文本提取
   - 只能渲染为图片显示

#### 04_ui_layer/01_architecture/README.md - UI架构

**源码位置**：`app/src/main/java/io/legado/app/ui/`

**核心内容**：

1. **Compose 声明式 UI**
   ```kotlin
   // 命令式
   textView.text = "Hello"
   textView.visibility = View.VISIBLE

   // 声明式
   Text("Hello")  // 根据状态描述 UI
   ```

2. **Screen/Content 分离**
   ```kotlin
   @Composable
   fun BookshelfScreen(viewModel: BookshelfViewModel = koinViewModel()) {
       val books by viewModel.books.collectAsState()
       BookshelfContent(books = books, ...)
   }
   ```
   - Screen：连接 ViewModel，处理状态订阅
   - Content：纯 UI 展示，支持 @Preview

3. **ViewModel 状态管理**
   ```kotlin
   class BookshelfViewModel : ViewModel() {
       val books: StateFlow<List<Book>> = repository.getAllBooks()
           .stateIn(viewModelScope, SharingStarted.Lazily, emptyList())
   }
   ```

4. **UiState 数据类**
   ```kotlin
   data class BookshelfUiState(
       val isLoading: Boolean = false,
       val errorMessage: String? = null,
       val showBookMenu: Book? = null
   )
   ```

5. **副作用处理**
   - `LaunchedEffect`：协程作用域
   - `SideEffect`：每次重组后执行
   - `DisposableEffect`：清理资源

6. **性能优化**
   - `remember` 缓存
   - `key` 优化列表
   - 延迟加载

#### 04_ui_layer/02_navigation/README.md - 导航系统

**源码位置**：`app/src/main/java/io/legado/app/ui/main/MainActivity.kt`

**核心内容**：

1. **NavHost 定义**
   ```kotlin
   NavHost(navController, startDestination = "bookshelf") {
       composable("bookshelf") { BookshelfScreen() }
       composable("book/{bookUrl}") { ... }
   }
   ```

2. **底部导航栏**
   ```kotlin
   NavigationBar {
       items.forEach { screen ->
           NavigationBarItem(
               selected = currentRoute == screen,
               onClick = {
                   navController.navigate(screen) {
                       launchSingleTop = true  // 避免重复
                       restoreState = true      // 恢复状态
                   }
               }
           )
       }
   }
   ```

3. **导航参数**
   - 定义：`arguments = listOf(navArgument("bookUrl") {...})`
   - 获取：`entry.arguments?.getString("bookUrl")`
   - URL 编码：特殊字符需编码

4. **Deep Link**
   ```xml
   <intent-filter>
       <data android:scheme="legado" android:host="book" />
   </intent-filter>
   ```

5. **ViewModel 与导航配合**
   - ViewModel 不持有 NavController（避免内存泄漏）
   - 通过 SharedFlow 发送导航事件
   - Screen 层执行实际导航
